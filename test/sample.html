ticker-tooltip 로컬 테스트

krx_map.js는 실제 배포와 동일하게 <script>로 로드합니다 (JSON fetch 아님). GAS만 로컬에 없으므로 TickerTooltip.fetchTickerData를 mock으로 교체해 검증합니다.

오늘 장에서 $삼성전자 와 $005930 은 같은 종목을 가리켜야 합니다.

중소형주 중에서는 $비에이치아이 가 눈에 띄었고, $에코프로비엠 은 보합이었습니다.

$없는종목 은 매핑에 없으므로 원문 그대로 남아야 합니다.

링크 안의 텍스트는 무시되어야 합니다: $삼성전자 링크텍스트

코드블록 안의 변수도 무시되어야 합니다:

const $삼성전자 = getPrice(); // 건드리면 안됨
확인 체크리스트
  • $삼성전자, $005930 → 둘 다 "삼성전자 ▲/▼N%" 뱃지로 변환되는지
  • $비에이치아이 → 하락(파랑) 뱃지, $에코프로비엠 → 보합(회색) 뱃지로 변환되는지
  • $없는종목 → 뱃지 없이 원문 그대로 남아있는지
  • <a> 안의 $삼성전자 → 뱃지로 바뀌지 않고 원문 그대로인지 (링크 텍스트 유지)
  • <pre>/<code> 안의 $삼성전자 → 뱃지로 바뀌지 않고 원문 그대로인지
  • PC: 뱃지에 마우스를 올리고 약 200ms 후 툴팁이 뜨는지, 벗어나면 닫히는지
  • 모바일 또는 개발자도구 기기 모드: 탭하면 툴팁이 뜨고, 다른 곳을 탭하면 닫히는지
  • 뱃지를 화면 가장자리로 스크롤한 뒤 열어서, 툴팁이 화면 밖으로 잘리지 않고 위치가 보정되는지
  • 한 번에 툴팁이 1개만 열리는지 (다른 뱃지를 열면 이전 툴팁이 닫히는지)
  • 툴팁 안 "네이버 금융에서 보기" 링크가 올바른 종목코드로 연결되는지
